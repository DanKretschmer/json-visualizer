🧰 Welcome to “Just JSON” — Learn JSON by Example

🎯 Purpose:
This site helps total beginners and early developers learn how to understand JSON and JSON Schema.

🧑‍🏫 What is JSON?

  • 🧱 JSON is a way to structure data using key-value pairs.
  • 📦 Data is organized into objects ({}) and arrays ([]).
  • 🧵 Keys are always strings; values can be strings, numbers, booleans, arrays, objects, or null.
  • 🔄 It looks a lot like JavaScript object syntax.
  • 🌐 It's used everywhere—from web APIs to config files.
  • 📄 JSON is plain text and easy for both humans and machines to read.
  • ⚙️ It's the standard for sending data between a browser and a server.

🛠️ What is a JSON Schema?

  • 📘 A JSON Schema is a set of rules for what a JSON object should look like.
  • 📐 It defines data types like string, number, object, array, and more.
  • 🔑 You can mark keys as required so they must be present in the data.
  • 🔁 $defs and $ref let you reuse parts of the schema to avoid repetition.
  • 🧪 Schemas help validate data before it's sent to or used by an app or API.
  • 📝 A template is like a filled-out version of a schema—an example of valid JSON based on the rules.
  • 🛠️ Use schemas to catch missing or wrong data early, especially in APIs and forms.

🧩 Key Concepts Made Simple

  • 🔑 properties define the keys inside your object — like the labels on a form.
  • ✔️ required means “don’t forget this key!” — it's not optional.
  • 🔠 type tells what kind of data you expect — like string, number, or object.
  • 🧱 Nesting means you can put objects inside other objects — like folders inside folders.
  • 📦 Arrays hold lists of values — use them when you need more than one of something.
  • 🚫 additionalProperties: false means “only the keys I listed are allowed” — no extras!
